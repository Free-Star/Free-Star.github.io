HTML入门

发表于2024-01-17|前端

定义

  • 标签符
    • 标签符一般都是成对出现的，包含一个“开始符号”和一个“结束符号”。结束符号只是在开始符号前面多加了一条斜杠“/”。当浏览器收到HTML文本后，就会解析里面的标签符，然后把标签符对应的功能表达出来。
    • <标签符>内容</标签符>
    • 标签也叫作“元素”，如p标签又叫p元素。叫法不同，意思相同。
  • HTML结构（如下）
    • 文档声明：
    • html标签对
    • head标签对
    • body标签对
  • 搜索蜘蛛
    • 在Web技术中，我们一般形象地称搜索引擎为“搜索蜘蛛”或“搜索机器人”。
  • HTML注释
    • 作用：注释
    • 语法：
    • ps：HTML标签是给浏览器看的，注释标签是给程序员看的。
  • 动态页面
    • 即使你的页面用了JavaScript，也不是动态页面，除非你还用到了后端技术。
  • 自闭合标签
    • 由于只有开始符号而没有结束符号，因此不可以在内部插入标签或文字。 所谓的“自闭合”，指的是本来要用一个配对的结束符号来关闭，然而它却“自己”关闭了。
  • 编程语言和标记语言有什么区别？
    • 编程语言是用于编写计算机程序的语言。它们提供了一套语法和规则，用于描述算法和数据处理的方式。编程语言通常具有丰富的功能和逻辑控制结构，可以进行条件判断、循环、函数定义等操作。编程语言可以用于开发各种类型的应用程序，从简单的脚本到复杂的软件系统。
    • 标记语言是用于描述文档结构和内容的语言。它们使用标记或标签来定义文本的结构、样式和语义。标记语言通常用于创建和格式化文档，如HTML用于创建网页，XML用于数据交换和配置文件等。标记语言主要关注于文档的结构和展示，通常没有编程语言那样的逻辑控制结构。

标签

结构标签

  • 文档声明
    • 是一个文档声明，表示这是一个HTML页面。
  • HTML标签
    • HTML标签的作用，是告诉浏览器，这个页面是从<html开始，然后到</html结束的。
  • head标签
    • head标签是网页的“头部”，用于定义一些特殊的内容，如页面标题、定时刷新、外部文件等。
    • 在HTML中，一般来说，只有6个标签能放在head标签内（如下）。
      • title标签
        • 唯一作用，定义标题
      • meta标签
        • 描述网页基本信息，是给“搜索蜘蛛”看的。
        • 重要属性
          • name
            • 描述网页的关键字等基本信息
            • keywords
            • description
            • author
            • copyright
          • http-equiv
            • 定义网页所使用的编码，定义网页自动刷新跳转
            • {meta http-equiv=”Content-Type” content=”text/html; charset=utf-8”/}
            • {meta http-equiv=”refresh” content=”6;url=http://www.lvyestudy.com"/}
            • {meta charset=”utf-8” /}这一句必须放在title标签以及其他meta标签前面
      • link标签
        • style标签用于定义元素的CSS样式
      • style标签
        • style标签用于定义元素的CSS样式
      • script标签
        • style标签用于定义元素的CSS样式
      • base标签
        • style标签用于定义元素的CSS样式
  • body标签
    • body标签是网页的“身体”，大部分代码都是在这个标签内部编写的。

文本标签

  • 标题标签

    • 在HTML中，共有6个级别的标题标签：h1、h2、h3、h4、h5、h6。其中h是header的缩写。

  • 段落标签

    • 用“p标签”来显示一段文字
    • 段落标签会自动换行，并且段落与段落之间有一定的间距。

  • 换行标签

    • br（break缩写）
    • 使用p标签会导致段落与段落之间有一定的间隙，而使用br标签则不会。

  • 文本标签

    • 粗体标签：strong、b
      • 尽量使用strong标签，而不要使用b标签。这是因为strong标签比b标签更具有语义性。
    • 斜体标签：i、em、cite
      • 因为语义性原因，尽量使用em标签。
    • 上标标签：sup
      • superscripted缩写
    • 下标标签：sub
      • subscripted缩写
    • 以下这几个标签几乎都不会使用，都会用css来实现
    • 中划线（删除线）标签：s
    • 下划线标签：u
    • 大字号标签：big
    • 小字号标签：small

  • 水平线标签

    • hr（horizon缩写），也是自闭合标签

  • div标签（分区标签）

    • division缩写
    • 使用div标签来划分区域，使得代码更具有逻辑性。
    • div标签最重要的用途是划分区域，然后结合CSS针对该区域进行样式控制。

  • 块元素

    • 块元素在浏览器显示状态下将占据整一行，并且排斥其他元素与其位于同一行。
    • 块元素内部可以容纳其他块元素和行内元素。

  • 行内元素

    • 行内元素与块元素恰恰相反，行内元素是可以与其他行内元素位于同一行的。
    • 行内元素内部（标签内部）只可以容纳其他行内元素，不可以容纳块元素。

  • 特殊符号(以&开头,以;结尾)

    • 空格符号
      • nbsp
        • 1个汉字约等于3个“nbsp”。
    • 其他符号
      • 难以通过输入法输入的需要用代码实现，都是以“&”开头，并且以“;”（英文分号）结尾的。

列表

  • 有序列表
    • ol (ordered list缩写)
      • ol标签的type属性，用来设置列表头格式，有五个值（1,a,A,i,I），都可以用css来实现
    • li (list缩写)
    • ol标签和li标签需要配合一起使用，不可以单独使用，而且ol标签的子标签也只能是li标签，不能是其他标签。
  • 无序列表
    • ul(unordered list缩写)
      • ul标签的type属性，用来设置列表头格式，有三个值（disc实心圆,circle空心圆,square正方形）
    • 与ol标签使用方式一样，也是必须得配合li标签使用
    • ul元素内部的文本，只能在li元素内部添加，不能在li元素外部添加。

HTML语义化

用某一个标签来代替另外一个标签实现相同的效果，大多数初学者都可能遇到过这种情况。正是这种错误思维，导致很多人在学习HTML时，没有认真地把每一个标签的语义理解清楚，糊里糊涂就学过去了。能用某一个学过的标签来代替，就懒得认真学新的标签，这是HTML学习中最大的误区。

语义化是非常重要的一个思想。 在整站开发中，编写的代码往往成千上万行，你现在的几行代码无法与其相提并论。如果全部使用div和span来实现，我相信你会看得头晕。要是某一行代码出错了怎么办？你怎么快速地找到那一行代码呢？除了可读性，语义化对于搜索引擎优化（即SEO）来说，也是极其重要的。
